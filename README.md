# 销售目标填报系统

一个简洁的销售目标收集网页应用，支持用户认证和数据隔离。

## 功能特性

- ✅ **用户登录**：通过销售ID和姓名登录
- ✅ **目标填报**：收集 GTV 目标和财收目标
- ✅ **数据隔离**：每个销售只能查看和编辑自己的数据
- ✅ **历史记录**：展示所有填报记录，支持删除
- ✅ **响应式设计**：适配桌面和移动设备

## 权限控制实现方式

### 1. 用户认证机制
- 使用 **localStorage** 存储当前登录用户信息
- 登录时保存 `salesId` 和 `salesName`
- 页面刷新后自动恢复登录状态

### 2. 数据隔离机制
```javascript
// 数据存储结构
{
  "targetData": {
    "S001": [  // 以 salesId 为 key
      { id: "xxx", gtv: 100, revenue: 50, createdAt: "..." }
    ],
    "S002": [  // 其他销售的数据完全隔离
      { id: "yyy", gtv: 200, revenue: 80, createdAt: "..." }
    ]
  }
}
```

### 3. 权限验证
- 所有数据操作都基于当前登录用户的 `salesId`
- 每个用户只能访问 `targetData[salesId]` 下的数据
- 未登录用户自动跳转到登录页

## 如何运行

### 方式一：直接打开（最简单）
1. 用浏览器直接打开 `index.html` 文件
2. 即可开始使用

### 方式二：使用本地服务器（推荐）
```bash
# 进入项目目录
cd sales-target-app

# 使用 Python 启动简单服务器
python3 -m http.server 8080

# 或使用 Node.js
npx serve .

# 或使用 VS Code Live Server 插件
```

然后访问 `http://localhost:8080`

### 方式三：部署到静态托管
将以下文件上传到任意静态托管服务：
- `index.html`
- `styles.css`
- `app.js`

支持的平台：GitHub Pages、Vercel、Netlify、阿里云 OSS 等

## 使用说明

1. **登录**：输入销售ID（如 S001）和姓名
2. **填报目标**：填写 GTV 目标和财收目标，点击提交
3. **查看记录**：在"我的目标记录"区域查看历史填报
4. **切换用户**：点击"退出"按钮，使用其他销售ID登录

## 技术栈

- **HTML5** - 页面结构
- **CSS3** - 样式和动画
- **原生 JavaScript** - 业务逻辑
- **localStorage** - 本地数据存储

## 注意事项

1. 数据存储在浏览器本地，清除浏览器数据会导致数据丢失
2. 如需持久化存储，需要接入后端数据库
3. 生产环境建议使用真实的用户认证系统（如 SSO、JWT 等）
